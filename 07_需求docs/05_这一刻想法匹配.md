# 05 · 这一刻想法匹配

> 文档目的：讲清楚「用户输入一句话 → 输出 3 条文案」的完整路径，含 AI / embedding 双模式、刷新会话、翻译。
> 涉及文件：`src/data/matchQuery.ts`、`src/services/chatchat/{ai,match,corpusIndex,matchSelect,matchSession}.ts`、`src/components/MatchedCopyModal.tsx`、`src/pages/Home.tsx`。

---

## 1. 入口

主页 `Home.tsx` 顶部输入框 → 用户写一句话（如「我承认我好色，但我也承认我想被认真对待」）→ 点匹配 → `matchQuotesByQuery(query)` → 弹 `MatchedCopyModal` 展示 3 条。

---

## 2. 双模式判定

```ts
const mode = isAiConfigured() ? "ai" : "embedding";
```

- `isAiConfigured()`：检查 `data/aiConfig.ts` 里是否配了 apiKey + baseUrl + model。
- 配了 → AI 模式：先调 LLM 解析意图再匹配。
- 没配 → embedding 模式：直接用离线向量算相似度。

用户在「我的」页可配置 OpenAI 兼容接口（baseUrl / apiKey / model）。

## 3. AI 模式

### 3.1 调用链

```
matchQuotesByQuery(query)
  → runQueryMatch(query, isRefresh=false)
  → rankForQuery(query, excludeIds)
    → parseUserIntent(query, apiConfig)        // 调 LLM
    → rankMatchesTwoStage(corpus, filters, query, excludeIds)
```

### 3.2 `parseUserIntent`

文件 `ai.ts`，向 `/v1/chat/completions` 发请求：

**System Prompt（摘要）**
```
你是朋友圈文案检索助手。用户描述想发的朋友圈情境，你只输出 JSON 检索条件，禁止生成任何文案。
规则：
1. mood/scene/style/purpose 各选 1~2 个最贴切的 tag_id
2. keywords 提取 3~5 个能在文案库检索的中文实体词
3. semantic_keywords 再补充 3~5 个同义/近义/联想词
4. 不要输出「朋友圈」「文案」等泛词
5. avoid 填写应排除的标签
tag_id 枚举：mood_*, scene_*, style_*, purpose_*, theme_*, length_*
输出纯 JSON：{"mood":[],...}
```

### 3.3 解析（`parseAiIntent`）

- 容错：AI 返回带 markdown 代码块时剥壳。
- `normalizeTags`：中文词经 `CN_TO_TAG` 转 tag_id，非 `xxx_yyy` 格式丢弃。
- `avoid` 标签从最终 filters 反向剔除。
- 关键字段：`mood / scene / style / purpose / theme / relation / length / keywords / semantic_keywords / avoid`。

### 3.4 排序（`rankMatchesTwoStage`）

```
1. searchCopies(corpus, filters, {limit:100, excludeIds}) → tagCandidates
2. buildSemanticTerms(userText, filters) → terms（含同义词扩展 SEMANTIC_SYNONYMS）
3. 对每个候选：
   semanticScore = computeSemanticScore(copy, userText, terms)
     = vocabScore * 0.7 + fullTextScore * 0.3
     - vocabScore：terms 在 text/keywords 命中率
     - fullTextScore：char-bigram Jaccard * 0.55 + 短语命中 * 0.45
   finalScore = tagScore * 0.3 + semanticScore * 0.7
4. finalizeRanked → 排序取前 RANK_POOL
```

### 3.5 同义词扩展（`SEMANTIC_SYNONYMS`）

`match.ts` 顶部硬编码约 20 组：
```
加班 → [打工, 上班, 职场, 工作, 累, 深夜]
工作 → [加班, 打工, 上班, 职场, 项目]
...
```
`buildSemanticTerms` 把用户词 + AI 给的 semantic_keywords 都扩展一遍，`textHasNegatedPositive` 会跳过被否定的正向词（「不开心」不扩展「开心」）。

## 4. Embedding 模式（无 AI）

### 4.1 调用链

```
matchQuotesByQuery(query)
  → runQueryMatch(query, false)
  → rankForQuery(query, excludeIds)
    → loadCorpusIndex()
    → rankMatchesByEmbedding(corpus, query, index, excludeIds)
```

### 4.2 `rankMatchesByEmbedding`

```
1. buildQueryVectorFromText(userText, index) → queryVec
   - char-bigram 入向量
   - 按标点切短语入 kw:
2. 对全语料（排除 excludeIds）算 cosineSimilarity
3. 过滤 > 0.01，排序，取前 RANK_POOL
```

这是最弱模式，完全靠 char-bigram TF-IDF 向量，**没有真正的语义理解**。
- 「我承认我好色」能匹配到 id=3669 是因为字面 + 标签重叠。
- 未来可换 bge 等真 embedding，详见 `03_匹配引擎.md` §4.5。

## 5. 刷新会话

### 5.1 流程

```ts
async function runQueryMatch(query, isRefresh) {
  const sessionKey = query.trim()
  let session = loadMatchRefreshSession(QUERY_MATCH_SESSION_KEY, sessionKey)
  if (isRefresh && session.refreshCount >= 5) {
    session = { sessionKey, shownIds: [], refreshCount: 0, baselineScore: undefined }
  }
  const ranked = await rankForQuery(trimmed, session.shownIds)
  const batch = selectRefreshBatch(ranked, 3, session.shownIds, 60, session.baselineScore)
  const topScore = batch[0]?.matchScore ?? ranked[0]?.matchScore
  session = commitMatchBatch(session, newIds, isRefresh, topScore)
  saveMatchRefreshSession(...)
  return { results, refreshRemaining, mode }
}
```

### 5.2 baseline 漂移防护

- 首次匹配：`baselineScore = batch[0].matchScore`
- 刷新：`selectRefreshBatch` 用 `baselineScore * 0.6` 绝对门槛过滤，低于的返回空。
- 空 → UI 提示「已无更多同主题」，不覆盖当前展示。

### 5.3 query 模式的剩余问题

`rankForQuery` 仍把 `session.shownIds` 传给 `rankMatchesTwoStage` / `rankMatchesByEmbedding`，候选池构建时排除已展示 → 池缩小。当前靠 baseline 门槛挡漂移，但理论上仍可能把无关条目拉到门槛之上。后续可同步去掉候选池的 excludeIds（与 quiz 模式一致），详见 `03` §8。

## 7. 翻译（`translateQuoteSemantically`）

`ai.ts` 另一个 LLM 调用，给单条文案做「语义解释」（不是字面翻译）：

**Prompt 摘要**
```
你是朋友圈文案的语义理解助手。用户会给你一条社交网络文案，可能中英日韩混排、带 emoji、歌词或名言引用。
请理解全文的语气、情境与意图，用自然流畅的中文帮读者弄懂「这句话想表达什么」。
输出要求：
1. 只输出中文正文，1～4 句，像朋友解释一样口语化
2. 外文按语境意译，保留原帖情绪
3. 原文已有清晰中文的部分不必复读
4. emoji 可保留不译
5. 禁止输出「翻译如下」等前缀
```

触发：`MatchedCopyModal` 里点某条文案的「翻译」按钮。
无 API 时该按钮禁用。
