## 阅读建议

- 第一次看：按 01 → 02 → 03 顺序，先建立整体地图再深入匹配引擎。
- 想调问卷：直接看 04，配合 02 的标签枚举。
- 想调匹配准确率：看 03，配合 05（query 模式）。
- 想加新页面 / 改 UI：看 06。
- 打包交给下一位：看 07，配合根目录 `README.md` 与 `04_设计资源/README.md`。

## 文档与代码同步原则

- 代码是唯一事实来源；文档与代码冲突时以代码为准，并请提醒我同步文档。
- 每次较大改动后，会在对应文档末尾的「后续可优化」清单里勾掉 / 新增条目。
- 文档里的文件路径都是相对项目根 `plus_projet/` 的。

## 当前重点待优化（跨文档汇总）

1. **向量索引升级**：char-bigram TF-IDF → 真 embedding（bge-small-zh 等），见 03 §4.5、05 §8。
2. **query 模式刷新漂移**：候选池仍排除 excludeIds，靠 baseline 门槛挡，建议与 quiz 模式一致改造，见 03 §8、05 §5.3。
1. **语料清洗已接 AI 二次复核**：`corpus_cleanup.py --ai` 调 DeepSeek 对规则扫描结果做判断，输出 `ai_verdict/ai_note`，最终 action 以 AI 为准。详见 02 §4.6。
4. **样式拆分**：`index.css` 3400 行单文件，组件增多后建议拆 CSS Module，见 06 §5.4。
5. **AI prompt few-shot**：降低中文不命中 tag_id 的概率，见 05 §8。
