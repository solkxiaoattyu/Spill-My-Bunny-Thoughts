## 1. 语料文件

主语料：`02_语料与数据/corpus/tagged_corpus.json`
当前规模：**≈4733 条**（清洗后；原 4753，删除 21 条噪声）
CSV 同步：`tagged_corpus.csv`（UTF-8-BOM，便于 Excel 查看）
构建期由 Vite 同时 serve 到 `/corpus/tagged_corpus.json`，前端 `loadCorpus()` fetch。

### 1.1 数据结构

```ts
interface CorpusCopy {
  id: number;
  text: string;
  tags: CopyTags;
  qc_passed?: boolean;
  qc_errors?: string[];
  qc_warnings?: string[];
}

interface CopyTags {
  mood?: string[];        // 情绪，可多选
  scene?: string[];       // 场景，可多选
  style?: string[];       // 风格，可多选
  purpose?: string[];     // 用途，可多选
  theme?: string[];       // 主题（怀旧/季节/友情…）
  relation?: string;      // 关系对象（单选字符串）
  length?: string;        // 长度（单选）
  tone_level?: number;    // 情绪强度 1-5
  tone_level_tag?: string;
  format?: string[];      // format_has_emoji / format_has_english ...
  content_rating?: string;// rating_clean / rating_mild_slang
  keywords?: string[];    // 打标时提取的关键词
}
```

### 1.2 加载与过滤

`src/services/chatchat/corpus.ts`：

```ts
function filterCorpus(raw: CorpusCopy[]): CorpusCopy[] {
  return raw.filter(
    (c) =>
      c.qc_passed !== false &&
      !FAILED_IDS.has(c.id) &&              // 与 qc_report.json 同步
      c.tags?.content_rating !== "rating_mild_slang",
  );
}
```

- `FAILED_IDS`：硬编码 QC 失败 id 集合（40 条左右），与 `qc_report.json` 手动同步。
- `content_rating === "rating_mild_slang"`：含轻俚语，前端默认隐藏。
- 加载后缓存在 `corpusCache`，全 App 共享一份。

|--------|------|-------------|

### 2.4 purpose（用途，权重 1.5）

| tag_id | 中文 | 问卷中归属 |
|--------|------|-----------|
| purpose_show_mood | 表达心情 | 发一条心情 |
| purpose_share_life | 晒生活 | 记录今天 |
| purpose_record | 仪式感 | 记录今天 |
| purpose_heal_self | 自我治愈 | 治愈一下自己 |
| purpose_cool_pose | 装酷 | 想发得酷一点 |
| purpose_humble_brag | 小得意 | 想发得酷一点（藏在「酷」下，不单独显眼） |
| purpose_subtle_love | 暗戳表白 | 暗戳戳表达喜欢 |

> 「低调炫耀」刻意不单独放显眼位置，避免用户不好意思点。

### 2.5 theme（主题，权重 1）

`theme_happiness / theme_love / theme_friendship / theme_work_life / theme_travel_wander / theme_self_love / theme_philosophy / theme_rainy / theme_season_spring / theme_season_autumn_winter / theme_night_mood / theme_pet / theme_foodie / theme_nostalgia / theme_sunny`

theme 不作为前台主选项，只在问卷的 scene / mood 选项里作为附带 tag 一起带出（例如选「出门旅行」会同时带 `scene_travel + scene_nature + theme_travel_wander`）。

### 2.6 relation（关系对象，权重 1，单选字符串）

| tag_id | 中文 | 触发场景 |
|--------|------|---------|
| relation_none | 不限定 | 默认 |
| relation_solo | 独处 | 选「治愈一下自己」后出现 |
| relation_crush | 暧昧对象 | 选「暗戳戳表达喜欢」后出现 |
| relation_couple | 情侣 | 选「暗戳戳表达喜欢」后出现 |

> **关系题是条件补充题**，不一开始就问；详见 `04_快速匹配问卷.md`。

### 2.7 length（长度，权重 0.5）

`length_ultra_short / length_short / length_medium / length_long`
语料里 ultra_short + short 占绝大多数，所以问卷默认推荐「一句话就够 / 两三句话」，`length_long` 不作为默认推荐。

## 3. 中文 → tag_id 映射

`labels.ts` 的 `CN_TO_TAG`：约 60 个常用中文词到 tag_id 的映射，用于：
- AI 意图解析返回中文时回填 tag_id（`ai.ts` 的 `normalizeTags`）。
- 离线兜底解析（`parseUserIntentLocal`，见 `ai.ts`）。

如果你新增了 tag_id，记得同步更新 `TAG_LABELS` 与 `CN_TO_TAG`，否则问卷/AI/历史标签会显示空。

## 4. 语料清洗流程

### 4.1 为什么要清洗

原始语料是从爬虫/合集里抓出来的，混了一批不适合直接当作文案本身的条目：
- 标题：「朋友圈文案｜出去玩这样发」「挑一句去发朋友圈吧」
- 水印：「勿二删二改」「封面搬运自潦草咪」
- 互动引导：「看到这篇笔记说明你已经在幸福里啦」
- 带货：「索尼1200W像素旗舰画质」「可复美胶原棒2.0」
- 自指开头：「我去我存了好多这种」「会一直更新哒」

这些条目如果直接被匹配出来，用户体验会很差。

### 4.2 清洗脚本

`01_前端应用/scripts/corpus_cleanup.py`

**用法**

```bash
# 1. 扫描生成待审核清单
python scripts/corpus_cleanup.py
# 输出 02_语料与数据/corpus/cleanup_candidates.json

# 2. 人工在 JSON 里改 action：
#    delete  — 删除（默认）
#    keep    — 保留
#    edit    — 改写，需填 new_text

# 3. 应用
python scripts/corpus_cleanup.py --apply
# 会同步更新 tagged_corpus.json 与 tagged_corpus.csv
# 之后需要 npm run build:index 重建向量索引
```

> Windows PowerShell 下中文路径可能编码乱码，建议用 `$env:CORPUS_DIR = "..."` 显式传语料目录。

### 4.3 规则定义（脚本顶部）

5 类正则数组：

| 类别 | 变量 | 示例规则 |
|------|------|---------|
| 标题/集合描述 | `TITLE_PATTERNS` | `朋友圈文案`、`文案｜`、`挑一句去发`、`适合.{0,8}这样发` |
| 水印/搬运 | `WATERMARK_PATTERNS` | `勿二删二改`、`勿搬运`、`封面搬运`、`侵删` |
| 互动/导流 | `PROMO_PATTERNS` | `看到这篇笔记说明你已经在`、`长按录制`、`求转赞评`、`公众号` |
| 品牌/带货 | `AD_PATTERNS` | `索尼\d+W像素`、`可复美`、`胶原棒`、`MUJI` |
| 自指/对话开头 | `SELF_REF` | `^我(去\|存了\|也有)`、`^会一直更新`、`^平常也喜欢码字` |

**加规则**：直接编辑对应数组即可，无需改逻辑。
**纯符号条目保留**：`SYMBOL_ONLY` 正则匹配到的纯 emoji/颜文字条目不删除（属于「符号装饰」风格）。

- 自指 5 条
- 品牌 4 条（含可复美 3 条 + MUJI 1 条）

### 4.6 AI 二次复核（已接入）

`corpus_cleanup.py --ai` 在规则扫描后，把候选条目分批（每批 20 条）送给 DeepSeek 判断。

**Prompt 设计（省 token）**
- System：定义「适合 vs 不适合」+ 输出 JSON 数组格式
- User：每条只送 `id + text[:60]`（截断长文本）
- `temperature=0`，`max_tokens=800`/批
- 每条 verdict 三选一：`keep / delete / edit`，note 限 20 字

**输出字段**
每条候选会补：
```json
{
  "id": 113,
  "text": "✏️📙可以自用喔|✋🏻勿搬运二改",
  "reason": "watermark",
  "reason_label": "水印/搬运声明",
  "ai_verdict": "delete",
  "ai_note": "水印声明非文案",
  "action": "delete"
}
```

**最终 action 取 AI verdict**（未复核时保留规则默认 delete）。你仍可手动改 action 覆盖 AI 判断。

**环境变量**
- `DEEPSEEK_API_KEY`：默认内置 `sk-a874...`，可覆盖。

**用法**
```bash
python scripts/corpus_cleanup.py --ai     # 扫描 + AI 复核
python scripts/corpus_cleanup.py --apply  # 应用 action
```

## 5. 离线向量索引（与语料配套）

`02_语料与数据/corpus/corpus_vectors.json`，由 `scripts/build-corpus-index.py` 生成。
内容：char-bigram + TF-IDF 稀疏向量，每条文案一个 `Map<termIdx, weight>`。
细节见 `03_匹配引擎.md`。

> **重要**：每次清洗语料后必须 `npm run build:index` 重建，否则向量索引和语料对不上。
