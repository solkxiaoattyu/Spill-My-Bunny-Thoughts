# 04 · 快速匹配问卷

> 文档目的：把 5 步问卷、条件关系题、懒人推荐卡、刷新不漂移的设计讲清楚。
> 涉及文件：`src/services/chatchat/quizConfig.ts`、`src/data/quiz.ts`、`src/components/{QuizModal,QuizResultsModal}.tsx`、`src/services/chatchat/{match,matchSelect,matchSession}.ts`。

---

## 1. 设计目标

用户打开问卷 → 5 步选择题 → 输出 3 条匹配文案 → 可换 5 批不重复。
不想一步步选的用户 → 顶部 8 张懒人卡，一键填好全部答案。

**文案要像人话**：不写「发布目的：表达心情」，写「我想发一条心情」。

---

## 2. 5 步问卷配置

定义在 `quizConfig.ts` 的 `QUIZ_STEPS`，顺序固定：**用途 → 场景 → 情绪 → 风格 → 长度**。

### 2.1 第一步：用途（purpose）

| 前台文案 | 对应 tag_id |
|---------|-------------|
| 发一条心情 | purpose_show_mood |
| 记录今天 | purpose_share_life + purpose_record |
| 治愈一下自己 | purpose_heal_self |
| 想发得酷一点 | purpose_cool_pose + purpose_humble_brag |
| 暗戳戳表达喜欢 | purpose_subtle_love |
| 吐槽一下 / 发疯一下 | purpose_complain |
| 想让朋友来互动 | purpose_interact |

> 「低调炫耀」（purpose_humble_brag）刻意藏在「想发得酷一点」下面，不单独显眼，避免用户不好意思点。

### 2.2 第二步：场景（scene）

| 前台文案 | 对应 tag_id |
|---------|-------------|
| 随手日常 | scene_daily + scene_universal |
| 出门旅行 | scene_travel + scene_nature + theme_travel_wander |
| 吃饭喝咖啡 | scene_food + theme_foodie |
| 自拍 / 状态照 | scene_selfie |
| 朋友聚会 | scene_social + theme_friendship |
| 一个人待着 | scene_home + theme_self_love |
| 工作学习 | scene_work + scene_study + theme_work_life |
| 夜晚 / 下雨 / 季节感 | scene_night + theme_rainy + theme_season_* |
| 宠物 | scene_pet + theme_pet |
| 音乐 / 演出 | scene_music |

> **「通用场景」（scene_universal）永远不显示这四个字**，前台叫「随手日常」。

### 2.3 第三步：情绪（mood）

| 前台文案 | 对应 tag_id |
|---------|-------------|
| 开心元气 | mood_happy + mood_energetic + theme_happiness |
| 温柔治愈 | mood_healing + mood_peaceful |
| 有点 emo | mood_emo |
| 反思人生 | mood_reflective + theme_philosophy |
| 幽默搞笑 | mood_humor |
| 浪漫心动 | mood_romantic + theme_love |
| 文艺氛围感 | mood_literary + mood_mysterious |
| 酷酷的 | mood_mysterious |

> **怀旧不在情绪主选项**，因为语料里这类标签数量很少，放到 theme 作为补充筛选。

### 2.4 第四步：风格（style）

| 前台文案 | 对应 tag_id |
|---------|-------------|
| 极简一句话 | style_minimal |
| 金句感 | style_quote |
| 像朋友说话 | style_colloquial |
| 诗意一点 | style_poetic |
| 小作文 | style_essay |
| 发疯文学 / 抽象一点 | style_abstract |
| 带 emoji / 符号装饰 | style_emoji_heavy + style_symbol_deco |
| 中英混排 | style_cn_en_mix |

> **「纯中文」（style_pure_cn）不作为前台选项**：用户一般不会主动说「我要纯中文」，但会说「不要英文」「不要 emoji」——后者用 avoid 表达。它仍在 AI prompt 与后台筛选里可用。

### 2.5 第五步：长度（length）

| 前台文案 | 对应 tag_id |
|---------|-------------|
| 一句话就够 | length_ultra_short |
| 两三句话 | length_short |
| 想认真写一段 | length_medium |
| 小作文模式 | length_long |

> 语料里 ultra_short + short 占绝大多数，默认推荐前两个；`length_long` 不作为默认推荐（库存少）。

## 3. 条件关系题（`QUIZ_RELATION_STEPS`）

**不一开始就问**，因为大多数文案是「无特定关系对象」。只在选了某些用途后才出现。

| 触发的用途 | 关系题选项 |
|-----------|-----------|
| purpose-2（治愈一下自己） | 写给自己（relation_solo） / 不限定对象（relation_none） |
| purpose-4（暗戳戳表达喜欢） | 写给暧昧对象（relation_crush） / 写给恋人（relation_couple） / 不想太明显（relation_none） |

### 3.1 数据结构

答案数组长度固定为 6 槽位：
- index 0~4：5 步必答题的 option key
- index 5：关系题答案（`QUIZ_RELATION_ANSWER_INDEX = 5`）

```ts
export const QUIZ_RELATION_ANSWER_INDEX = 5;

export function getRelationStep(purposeKey: string): QuizStep | null {
  return QUIZ_RELATION_STEPS[purposeKey] ?? null;
}
```

### 3.2 切换用途时清空关系题

`QuizModal` 里 `selectOption(0, key)` 会调 `clearRelationAnswer(next)`，把 index 5 清空，避免选了「暧昧」再切「治愈」后还残留暧昧的关系答案。

## 4. 懒人推荐卡（`QUIZ_PRESET_CARDS`）

8 张，定义在 `quizConfig.ts`：

| 卡片名 | 筛选逻辑（答案组合） |
|--------|---------------------|
| 今日朋友圈随手发 | 记录今天 + 随手日常 + 开心元气 + 极简 + 一句话 |
| 温柔治愈一下 | 治愈自己 + 随手日常 + 温柔治愈 + 诗意 + 一句话 + 关系:不限定 |
| 适合 plog 的文案 | 记录今天 + 随手日常 + 开心元气 + emoji符号 + 一句话 |
| 暧昧但不明说 | 暗戳戳 + 随手日常 + 浪漫心动 + 口语 + 一句话 + 关系:暧昧对象 |
| 有点发疯但好笑 | 吐槽 + 随手日常 + 幽默搞笑 + 抽象 + 一句话 |
| 适合自拍状态照 | 酷一点 + 自拍 + 酷酷的 + 极简 + 一句话 |
| 旅行路上发什么 | 记录今天 + 出门旅行 + 文艺氛围 + 诗意 + 两三句 |
| 今天有点 emo | 发心情 + 随手日常 + 有点emo + 金句 + 一句话 |

### 4.1 数据结构

```ts
interface QuizPresetCard {
  id: string;
  label: string;
  subtitle: string;
  answers: Record<string, string>;  // stepId → optionKey
}
```

### 4.2 应用逻辑

`answersFromPreset(preset)`：把 `answers` 6 槽位按 stepId 填好。
- `purpose` → index 0
- `scene` → index 1
- `mood` → index 2
- `style` → index 3
- `length` → index 4
- `relation` → index 5

`QuizModal` 里点卡片 → `applyPreset(id)` → `setAnswers(answersFromPreset(preset))` + `setActivePresetId(id)` 高亮。

## 5. 答案 → filters → 匹配

### 5.1 答案结构

```ts
answers: string[]   // 长度 6
// answers[0] = "purpose-2"  ← 治愈一下自己
// answers[1] = "scene-0"    ← 随手日常
// answers[2] = "mood-1"     ← 温柔治愈
// answers[3] = "style-3"    ← 诗意一点
// answers[4] = "length-0"   ← 一句话就够
// answers[5] = "relation-1" ← 不限定对象
```

### 5.2 转 filters（`answersToContext`）

```ts
for (i = 0..4) {
  option = QUIZ_QUESTIONS[i].options.find(key === answers[i])
  selections.push({ tags: option.tags })
  labels.push(option.label)   // 给向量查询用
}
if (answers[5]) {
  rq = getActiveRelationQuestion(answers)
  option = rq.options.find(key === answers[5])
  selections.push({ tags: option.tags })
  labels.push(option.label)
}
filters = mergeQuizAnswers(selections)
```

### 5.3 匹配调用

```ts
const ranked = rankMatchesQuizTwoStage(corpus, filters, labels, index);
const batch = selectRefreshBatch(ranked, 3, session.shownIds, 60, session.baselineScore);
```

详见 `03_匹配引擎.md`。

## 6. 刷新不漂移（重点）

### 6.1 问题现象

用户选「暧昧文案」，刷新 2 次后变成「文艺旅游文案」。

### 6.2 根因

1. `rankMatchesQuizTwoStage` 原来把 `excludeIds` 传给 `searchCopies`，已展示 id 被排除 → 候选池缩小 → 触发 `RELAX_ORDER` 渐进放宽（依次丢掉 purpose→length→style→scene）→ 最后只剩 mood + 向量语义 → 无关条目被拉上来。
2. `selectRefreshBatch` 用「当前池最高分 × 60%」作为门槛，候选变差时门槛也跟着降，无关条目照样合格。

### 6.3 修复

| 改动点 | 文件 | 改法 |
|--------|------|------|
| 候选池不排除 excludeIds | `match.ts` `rankMatchesQuizTwoStage` | `searchCopies(corpus, filters, {limit:100})`，不传 excludeIds |
| 引入 baseline 绝对门槛 | `matchSession.ts` | `MatchRefreshSession` 加 `baselineScore`，首次匹配时记录 |
| 刷新用绝对门槛 | `matchSelect.ts` `selectRefreshBatch` | 有 baseline 时用 `baselineScore * 0.6` 过滤，不达标返回空 |
| 空结果 UI 提示 | `QuizResultsModal.tsx` | 检测空结果显示「已无更多同主题高匹配文案」，不覆盖当前展示 |
| 重新开始换批 | `QuizResultsModal.tsx` | 清 session + 缓存 → 重新首次匹配重建 baseline |

### 6.4 效果

- 选「暧昧」刷新 5 批，每批都是暧昧/浪漫相关。
- 同主题高匹配文案不够 5 批时，提前提示「已无更多」，不硬塞无关条目。
- 点「重新开始换批」从首轮重新挑。

## 7. UI 细节

### 7.1 `QuizModal.tsx`

- 顶部：8 张懒人卡 2 列网格，点击高亮 + 填答案
- 中部：5 步表单，每步一个 section（左侧 SVG icon + 分类名 + hint，右侧问题 + tag 按钮）
- 关系题：选完用途后动态插入，淡黄底色区分
- 进度条：节点数 = 5 + (有关系题 ? 1 : 0)
- 提交按钮：`isQuizReadyToSubmit` 要求前 5 步全填

### 7.2 `QuizResultsModal.tsx`

- 复用 `MatchedCopyModal` 展示 3 条结果
- 「换一批」按钮 + 剩余批次提示
- 空结果时按钮变「重新开始换批」，提示文案变化
- 每次刷新调 `recordMatchHistory` 记录历史

## 8. 后续可优化

- [ ] 懒人卡当前是固定 8 张，可按用户最近匹配历史动态生成「你上次选过…」
- [ ] 关系题触发条件目前硬编码在 `QUIZ_RELATION_STEPS`，可改成 `showWhen` 表达式（如选了「朋友聚会」也问关系）
- [ ] 5 步全选完才能提交，可加「跳过某步」的弱约束（除用途外都允许跳过，跳过维度不参与 filters）
- [ ] 进度条节点点击可跳转该步，目前只是展示
- [ ] 懒人卡点击后不自动滚动到底部提交，用户可能想微调，当前设计是对的，但可加「长按卡片直接匹配」快捷路径
