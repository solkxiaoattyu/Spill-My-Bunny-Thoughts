# 07 · 交接与打包说明

> 给下一位接手开发者：如何跑起来、改什么、打包交付。

## 1. 项目包结构（plus_projet）

| 目录 | 作用 |
|------|------|
| `01_前端应用/` | React 主工程，日常开发在这里 |
| `02_语料与数据/` | 语料 JSON / CSV / 向量索引 |
| `03_数据打标后端/` | 离线 Python 打标流水线 |
| `04_设计资源/` | 图片源文件 + 同步脚本（见该目录 README） |
| `05_旧版Web原型/` | 早期 HTML 原型，仅供参考 |
| `06_构建产物_dist/` | 生产构建快照，可静态部署 |
| `docs/` | 设计与实现文档 |

## 2. 本地跑起来

```bash
cd 01_前端应用
npm install
npm run dev
# http://localhost:5173/home
```

若图片 404，先同步设计资源：

```bash
cd ../04_设计资源
python sync_to_frontend.py
```

## 4. 打包给下一个人

### 推荐打包内容

- 整个 `plus_projet/` 文件夹
- **可省略**（体积大、可重装）：
  - `01_前端应用/node_modules/`
  - `01_前端应用/dist/`（若已提供最新的 `06_构建产物_dist/`）

### 打包前检查清单

- [ ] `python 04_设计资源/sync_to_frontend.py` 已执行，`01_前端应用/public/` 资源齐全
- [ ] `cd 01_前端应用 && npm run build` 成功
- [ ] `06_构建产物_dist/` 已更新（见下方命令）
- [ ] 无 `.env`、API Key 等敏感信息被打包

### 更新构建产物到 06 目录

```bash
cd 01_前端应用
npm run build
cd ..
python tools/sync_build_dist.py
```

### 仅预览构建结果（不接开发）

```bash
cd 06_构建产物_dist
npx serve .
```
